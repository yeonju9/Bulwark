# Phase 5 — 폴리시 (상용 퀄리티 만들기)

> 목표: "개인 프로젝트"와 "서비스되는 게임"의 차이를 메운다.
> 규모: 중간~큼. 선행: Phase 2~4. (Phase 4와 일부 병행 가능)

## 1. 설계 결정

### 1-1. AI 아이콘 파이프라인 (유료 에셋 없음)
- **스타일 가이드 프롬프트를 하나 고정**하고 모든 아이콘에 재사용 — 일관성이 생명.
  예: "fantasy game item icon, painterly style, dark background, centered,
  soft rim light, 512x512, no text" + 아이템별 묘사
- 생성 도구: Gemini 이미지 생성 / Claude 시각 자료 활용. **같은 도구로 전부 생성** (혼용 금지)
- 후처리: 512 → webp 128px 일괄 변환 스크립트 (`scripts/icons.mjs`, sharp 사용)
- 파일 규약: `apps/web/public/icons/{itemId}.webp` — **데이터의 id가 곧 파일명**
- 코드 변경: `ItemDef.icon`(이모지)을 유지하되 `<ItemIcon id={...}>` 컴포넌트가
  이미지 존재 시 이미지, 없으면 이모지 폴백 → 점진 교체 가능
- 필요 수량 추산: 아이템 ~60 + 스킬 8 + 몬스터 ~12 + UI = **약 100장**

### 1-2. 사운드
- 라이선스: CC0만 (freesound.org, kenney.nl 오디오 팩) — 출처 `CREDITS.md`에 기록
- 종류 최소화: 액션 완료(스킬군별 1개), 레벨업, 전투 타격, 던전 클리어, UI 클릭 = ~10개
- 구현: Web Audio 또는 howler.js. **음소거 기본 OFF 아님** — 첫 상호작용 후 재생(브라우저 정책)

### 1-3. 온보딩
- 모달 튜토리얼 금지. **목표 카드 방식**: 화면 상단에 "첫 목표: 일반 나무 베기 10회" →
  완료 시 다음 목표 (제련 → 검 제작 → 장착 → 슬라임 사냥). 5~6개로 핵심 루프 안내
- 신규 유저는 사이드바를 벌목만 보이게 시작? → 아니오. 잠금 표시는 유지하되 목표 카드로 유도 (단순하게)

### 1-4. PWA / 모바일
- `vite-plugin-pwa`: 설치 가능 + 오프라인 셸 (게임 자체가 오프라인 동작하므로 궁합 좋음)
- 모바일 레이아웃: 사이드바 → 하단 탭바 전환 (breakpoint 768px)
- 이것이 "모바일 RPG" 원래 목표와 만나는 지점 — 웹뷰 래핑/스토어 등록은 백로그

## 2. 데이터 모델 변경
- 세이브 v5: `settings: { muted: boolean; lang: 'ko' | 'en' }`, `tutorialStep: number`
- `statistics: Record<string, number>` (누적 채집/처치/골드 — 통계 화면 + Phase 7 업적 대비)

  안 되면 차선책: 단색 실루엣 스타일(생성 난이도 낮음)로 방향 전환
