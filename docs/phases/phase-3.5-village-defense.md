# Phase 3.5 — 마을 방어 전환 (맵·건설·웨이브)

> 목표: 게임의 중심을 "캐릭터 성장"에서 **"마을을 지키는 것"**으로 전환.
> 화면 중앙에 맵, 마을 3×3 건설, 자동 웨이브 침공, 맵 위 던전.
> 상태: **설계 합의 완료 (2026-06-13), 구현 미착수.** UI 테마 시안 결정 대기.
> 선행: Phase 3. Phase 4(계정·서버)보다 먼저 진행.

## 1. 확정된 설계 결정 (사용자 합의)

### 1-1. 맵과 마을
- 메인 화면 중앙 = **맵**. 단계별로 여러 맵 예정, 우선 1개만 구현
- 1단계 맵 이름: **"시작의 마을"** — 잔디밭이 들어간 평화로운 느낌
- 맵 안에 캐릭터가 지켜야 할 **마을(3×3 그리드)**. 1칸에 건물 1개만 건설
- 기존 캐릭터/사냥터/던전 패널은 맵 화면으로 통합. 스킬·인벤토리·상점 패널 유지
- **유저의 궁극 목표 = 마을 방어**

### 1-2. 마을 스탯 (캐릭터 스탯 대체)
- 캐릭터가 아닌 **마을이 HP·공격력·방어력**을 가짐
- 구성: **건물 주력 + 공격/체력 스킬 레벨 보조 + 장비 보조**
  (무기→마을 공격력, 방어구→마을 방어력. 기존 성장·제작 루프 전부 유지)
- 기존 스킬 8종은 그대로 유지
- HP 자연 회복 유지. 기존 전투 음식 → "보급품"으로 재해석 (마을 복구)

### 1-3. 건물 (시작은 3종)
- **본부**: 기본 HP/공/방 제공. 시작 시 건설된 상태. 파손되지 않음
- **성벽**: 최대 HP + 방어력
- **병영**: 공격력
- 같은 건물 중복 건설 가능(효과 누적). 비용 = 골드 + 자원(원목/주괴 등)
  → 골드·제작품 사용처 추가

### 1-4. 웨이브 침공 (자동 수비 시스템)
- 일정 주기마다 몬스터가 **자동으로** 마을을 공격. 유저 시작 버튼 없음
- **작업 슬롯을 차지하지 않음** — 방어는 상시 자동, 채집/제작과 경쟁하지 않음
- 마을 스탯으로 자동 전투 (기존 결정적 기대값 전투 `settleHunt` 로직 재사용,
  웨이브 번호 = 경과 시간의 닫힌 식 → 오프라인 정산 동일성 유지)
- **난이도(티어)는 던전 최초 클리어 수로 해금.** 던전을 안 깨면 난이도 유지
- 유저는 해금된 범위 안에서 **티어를 직접 선택** 가능 (낮춰서 안전 운영 가능)
- **난이도 ∝ 보상** (전리품·XP 배율)
- 승리: 전리품 + 공격/체력 XP. 패배(마을 HP 부족): **건물 1개 파손**
  (우선순위: 성벽 → 병영, 본부 제외) 후 농성 모드(피해·보상 없음)
- 파손 건물: 효과 정지, 골드+자원으로 수리
- **오프라인: 웨이브 진행하되 보상 50% 정산, 파손은 정산당 최대 1회**
  (50% = "보상 50%"로 해석 — 사용자 재확인 필요 시 여기 갱신)

### 1-5. 던전 (수동 공략 시스템 — 웨이브와 완전 별개)
- 맵 위 마을 주변에 레벨별 고정 배치 (시안: Lv5/12/20/30 네 곳)
- **던전 전용 몬스터를 따로 추가 배치** (웨이브 몬스터 풀 재사용 아님).
  같은 레벨대 웨이브 몬스터보다 강함
- **유저가 원할 때 언제든 입장** — 기존 10분 쿨다운 제거
  (반복 보상이 소량이라 악용 여지 적음)
- 즉시 판정 (기존 `attemptDungeon` 방식, 마을 수비대 파견 컨셉, 마을 스탯 사용)
- **최초 1회 클리어만 큰 보상** + 다음 웨이브 티어 해금. 이후 반복은 소량 보상
- 웨이브와의 유일한 연결고리: 최초 클리어 → 티어 해금

## 3. 구현 순서 (미착수)

1. **코어 v5**: `village { hp, buildings[9] }`, `waveTier`, `dungeonClears` + v4→v5 마이그레이션
   (기존 `hp`→마을 HP 환산, 진행 중 사냥 작업 제거, goblin_den 보스 처치 이력 → 클리어 1회 인정)
2. **코어**: `computeVillageStats` + 건물 데이터 3종 (건설/수리 커맨드)
3. **코어**: 웨이브 정산 `settleWaves` — simulate 통합, 파손/농성/티어/오프라인 50% 규칙
4. **코어**: 던전 최초/반복 보상 이원화 + 티어 해금 + 쿨다운 제거 + 던전 전용 몬스터 추가
5. **데이터**: 맵 1 "시작의 마을" 정의 (티어별 웨이브 몬스터 테이블, 던전 4개·배치)
6. **UI**: `MapPanel`(기본 화면), TopBar 마을 HP, 사이드바 정리, 확정 테마 적용
7. **검증**: 테스트 갱신 + 웨이브/파손/티어 테스트, `npm run balance`, 본 문서에 진행 기록

밸런스 수치(웨이브 주기, 건설 비용, 티어 배율 등)는 구현하며 balance 스크립트로 잡고 여기에 기록.
