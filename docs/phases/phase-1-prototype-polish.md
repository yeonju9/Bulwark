# Phase 1 — 프로토타입 다듬기 (재미 검증)

> 목표: "숫자가 올라가는 게 재미있는가"에 답한다. 신규 시스템 없이 피드백과 밸런스만.
> 규모: 작음 (며칠). 선행: Phase 0 ✅

## 1. 설계 결정

| 결정 | 권장안 | 이유 |
|---|---|---|
| 알림 방식 | 우상단 토스트 스택 (3초 자동 소멸, 최대 4개) | 모달은 흐름을 끊음 |
| 알림 데이터의 출처 | `Gains`에 이미 있는 levelUps + 슬롯 해금은 simulate 전후 비교 | 코어 변경 최소화 |
| 밸런스 목표 곡선 | 해금 간격: 초반 5~10분 → 중반 수 시간 → 후반 1일+ | 방치형 표준 페이싱 |

## 2. 데이터 모델 변경

세이브 스키마 변경 **없음** (v2 유지). 코어 변경도 최소:
- `Gains`는 그대로 사용. 토스트는 웹 레이어에서 `tick()` 반환값을 구독해 생성.

## 3. 작업 분해 (순서대로)

### 3-1. 레벨업/해금 토스트 (P0)
- [ ] `web/src/toast.ts` — 토스트 스토어 (zustand 또는 같은 스토어에 `toasts: Toast[]`)
- [ ] `store.tick()`에서 `gains.levelUps`가 있으면 토스트 push.
      슬롯 해금: tick 전후 `unlockedActionSlots()` 비교 → "작업 슬롯 해금! (2/3)"
- [ ] 액션 해금 감지: 레벨업 시 해당 스킬의 `actionsForSkill` 중
      `levelRequired === 새 레벨`인 액션 → "참나무 벌목 해금!"
- [ ] `components/Toasts.tsx` + CSS (slide-in, fade-out)

### 3-2. 밸런스 1차 패스 (P0)
- [ ] `packages/core/scripts/balance-report.ts` — 헤드리스 시뮬레이터.
      각 스킬에 대해 "최적 액션만 돌렸을 때 레벨 n 도달까지 걸리는 시간" 표 출력
      (코어를 노드에서 직접 실행 — 공유 코어 구조의 이점을 처음 써먹는 곳)
- [ ] 출력 예: `벌목 Lv10(참나무 해금): 42분` — 전 해금 포인트 나열
- [ ] 목표 곡선과 비교해 `data/skills.ts` 수치(durationMs/xp) 조정
- [ ] 판매가 점검: 시간당 골드가 티어에 비례해 증가하는지
- [ ] 슬롯 해금(총 레벨 30/90) 도달 시간 확인 — 30은 첫 세션(1~2시간) 내 권장

### 3-3. 세이브 내보내기/가져오기 (P1)
- [ ] `web/src/save.ts`에 `exportSave(): string` (JSON), `importSave(text): GameState | null` (migrateSave 재사용)
- [ ] 설정 패널 신설 (`components/SettingsPanel.tsx`, 사이드바에 ⚙️ 추가)
      — 내보내기(클립보드/파일 다운로드), 가져오기(텍스트 붙여넣기), 초기화
- [ ] 가져오기 실패 시 기존 세이브 유지 + 에러 토스트

### 3-4. 진행 초기화 (P1)
- [ ] 설정 패널 안. 2단계 확인 ("정말로?" → "되돌릴 수 없습니다" 텍스트 입력 or 더블 확인)

### 3-5. QoL (P2)
- [ ] 액션 카드에 시간당 획득량: `3600000/durationMs × qty` (아이템/시), XP/시
- [ ] `document.title` 갱신: 진행 중 작업 표시 `[벌목 중] Idle RPG`
- [ ] 레벨업 시 사이드바 레벨 숫자 잠깐 강조 애니메이션

## 4. 테스트 계획
- 코어 변경이 없으므로 신규 코어 테스트는 없음
- balance-report 스크립트가 곧 밸런스 회귀 테스트 역할 (수치 바꿀 때마다 재실행)
- 수동: 새 세이브로 30분 플레이 — 해금이 끊기지 않는지 체크

## 5. 완료 기준 (DoD)
- [ ] 30분 연속 플레이에서 "다음 목표"가 끊기지 않는다
- [ ] 레벨업·액션 해금·슬롯 해금에 토스트가 뜬다
- [ ] 세이브를 파일로 백업하고 복원할 수 있다
- [ ] balance-report 표가 목표 곡선(초반 5~10분 간격)과 대체로 일치한다

## 6. 리스크 / 주의
- 밸런스에 빠져 시간을 너무 쓰지 말 것 — "1차 패스"다. 전투(Phase 2)가 들어오면 어차피 다시 잡는다.
- 토스트가 오프라인 정산 모달과 동시에 뜨지 않게 — 모달 표시 중에는 토스트 억제.
