# Phase 2 — 전투와 던전 (게임의 척추)

> 목표: 채집→제작→**전투**→더 나은 채집의 순환 고리 완성. RPG가 되는 단계.
> 규모: 큼 (수 주). 선행: Phase 1. **이 프로젝트에서 가장 중요한 Phase.**

## 1. 설계 결정

### 1-1. 전투 모델: 결정적 기대값 전투 + 시드 난수 전리품
- **전투 수치는 난수 없이 기대값으로 계산한다.** 명중률을 굴리는 대신
  `실효 DPS = 공격력 × 명중률(공격 vs 방어)`로 처치 시간을 닫힌 식으로 구한다.
  → 오프라인 정산이 온라인과 항상 동일 (운영 원칙 2: 코어 결정성)
- **전리품만 난수**, 단 시드 주입: `mulberry32(seed)`, 시드 = `lastTickAt ^ 처치 누계`.
  같은 (상태, 시각)이면 같은 전리품 → simulate 재실행 검증(Phase 4)과 호환.

### 1-2. 전투도 "작업"이다
- 사냥 = 액션 (사이클 = 몬스터 1마리 처치). 기존 simulate/슬롯 구조 재사용.
- 단, **전투 작업 중에는 다른 전투 작업 불가** (스킬 단위 제한이 자동으로 보장).
- 사이클 시간이 고정이 아니라 스탯 의존 → 코어에 `cycleDurationMs(action, state)` 도입.

### 1-3. 전투 스킬과 스탯
- 신규 스킬 2종: **공격**(전투 XP, 데미지·명중에 기여), **체력**(피격 시 XP, 최대 HP에 기여)
  → 스킬 5종이 됨. 슬롯 해금 총 레벨(30/90)은 유지하되 balance-report로 재검증.
- 파생 스탯 (순수 함수 `computeStats(state)`):
  - `공격력 = 기본 + 공격레벨 보너스 + 무기 공격력`
  - `방어력 = 기본 + 장비 방어력 합`
  - `최대HP = 10 × 체력레벨`
  - `명중률 = clamp(공격측 명중 / (공격측 명중 + 수비측 방어), 0.2, 0.95)` — 1차안, 시뮬레이터로 조정

### 1-4. 사망 페널티: 소프트
- HP가 모자라면 **그 시점에 사냥이 멈출 뿐** (아이템/XP 손실 없음).
- 던전 실패 = 보상 없음. 그것으로 충분한 긴장감. 하드 페널티는 방치형에서 이탈 요인.

### 1-5. HP 회복과 음식
- 비전투 시 HP 자동 회복 (분당 체력레벨 비례).
- 사냥 작업에 **음식 슬롯**: 지정한 음식을 자동 섭취 (HP < 50%에 1개).
  Phase 2에서는 임시 음식(몬스터 드랍 '말린 고기')으로 시작, Phase 3 요리가 본 공급처.

## 2. 데이터 모델 변경 (세이브 v3)

```ts
// types.ts 추가
export type SkillId = ... | 'attack' | 'hitpoints';
export type EquipSlot = 'weapon' | 'armor';

export interface MonsterDef {
  id: string; name: string; icon: string;
  levelRequired: number;        // 공격 레벨 요구치
  hp: number; attack: number; defense: number;
  attackIntervalMs: number;     // 몬스터 공격 주기
  lootTable: LootEntry[];       // { itemId, qty, weight, chance }
  xp: number;                   // 처치당 공격 XP
}

export interface DungeonDef {
  id: string; name: string;
  monsters: string[];           // 연전 순서, 마지막이 보스
  rewardTable: LootEntry[];     // 클리어 보상 (티어 해금 재료)
}

// GameState 추가 필드
equipment: Record<EquipSlot, ItemId | null>;
hp: number;                          // 현재 HP (정산 시점 기준)
combatFood: ItemId | null;           // 사냥 음식 슬롯
monsterKills: Record<string, number>; // 도감용 처치 누계

// ItemDef 확장
equip?: { slot: EquipSlot; attack?: number; defense?: number };
food?: { heal: number };
```

마이그레이션 v2→v3: 신규 필드 기본값 채우기 (`equipment: {weapon:null, armor:null}`, `hp: 최대HP`, ...).

## 3. 작업 분해 (순서대로)

### 3-1. 코어: 스탯과 장비 (전투 없이 먼저)
- [ ] `core/src/combat/stats.ts` — `computeStats(state)` 순수 함수 + 단위 테스트
- [ ] 장비 커맨드: `equipItem(state, itemId)` / `unequipItem(state, slot)`
      (인벤토리 ↔ 장비 슬롯, 착용 중 아이템 판매 방지)
- [ ] 기존 검 3종에 `equip` 스탯 부여 + 방어구 아이템·제작 레시피 추가 (가죽 갑옷 등)
- [ ] 세이브 v3 마이그레이션 + 테스트

### 3-2. 코어: 가변 사이클 시간
- [ ] `cycleDurationMs(action, state)` — 일반 액션은 `action.durationMs`,
      전투 액션은 `ceil(몬스터HP / 실효DPS)` 기반
- [ ] simulate가 이 함수를 쓰도록 변경. **주의**: 사이클 중 레벨업으로 DPS가 변해도
      해당 정산 구간 안에서는 시작 시점 스탯 고정 (단순화, 문서화)

### 3-3. 코어: 사냥 시뮬레이션
- [ ] `core/src/combat/hunt.ts` — 사이클당: 처치 시간, 받는 피해(몬스터 DPS × 노출 시간),
      음식 자동 섭취, 공격/체력 XP, 시드 전리품
- [ ] 정지 조건: `hp - 사이클 피해 < 0`이고 음식 없음 → `stopped: 'low-hp'`
- [ ] 오프라인 정산: 닫힌 식으로 "음식과 HP로 버틸 수 있는 최대 사이클 수" 계산
      (사이클 루프 돌리지 말 것 — 12시간 정산 성능)
- [ ] 몬스터 5~8종 데이터 (`data/monsters.ts`): 슬라임 → 늑대 → 고블린 → 오크 → … 티어 구성
      전투 전용 드랍: 가죽(방어구 재료), 마정석(Phase 3 연금술 예약), 말린 고기(임시 음식)
- [ ] 비전투 HP 자동 회복을 simulate에 통합

### 3-4. 코어: 던전
- [ ] 던전 = 즉시 판정 커맨드 `attemptDungeon(state, dungeonId, now)`:
      몬스터 순서대로 기대값 전투 → 총 소요시간/총 피해 계산 →
      성공 시 보상 + 시간 경과, 실패 시 어디서 죽었는지 반환
      (방치 루프가 아니라 "준비해서 도전"하는 단발 콘텐츠 — 전략 포인트)
- [ ] 던전 1개: 「고블린 소굴」(잡몹 3 + 보스). 보상: 미스릴 강화석(다음 티어 제작 잠금 해제 재료)
- [ ] 던전 입장 조건: 진행 중 사냥 작업 없음

### 3-5. 웹 UI
- [ ] 사이드바: 전투 섹션 (공격/체력 스킬 + 사냥터 + 던전)
- [ ] 사냥 패널: 몬스터 카드 (스탯, 예상 처치시간, 예상 손익 — "시간당 피해 > 회복"이면 경고)
- [ ] 캐릭터 패널: 장비 슬롯, 스탯, HP 바, 음식 슬롯 지정
- [ ] 던전 패널: 입장 → 결과 리포트 모달 (전투 로그 요약)
- [ ] 도감 패널: 몬스터 처치 수 / 발견 아이템

### 3-6. 밸런스
- [ ] balance-report 확장: 장비 티어별 vs 몬스터 티어별 승률/손익표
- [ ] "구리 장비로 슬라임은 안전, 늑대는 음식 필수, 고블린은 무리" 같은 의도 곡선 맞추기

## 4. 테스트 계획 (코어)
- computeStats: 장비/레벨 조합별 스냅샷
- 사냥 정산: 온라인(틱 누적)과 오프라인(일괄)이 같은 결과 — **가장 중요한 테스트**
- 음식 소진 → low-hp 정지, 음식 충분 → 계속
- 시드 전리품: 같은 (상태,시각) → 같은 드랍
- 던전: 성공/실패 경계 케이스, 보상 지급
- v2→v3 마이그레이션

## 5. 완료 기준 (DoD)
- [ ] 만든 장비를 끼고 던전 보스를 잡으면 다음 티어 재료가 나온다
- [ ] 준비 부족으로 전투에서 패배할 수 있다 (전략이 존재한다)
- [ ] 오프라인 사냥 정산이 온라인과 같은 결과를 낸다 (테스트로 보장)
- [ ] 전투 전용 드랍에 제작 소비처가 있다 (가죽→방어구)

## 6. 리스크 / 주의
- **스코프 경보**: 몬스터 종류·던전 수를 늘리고 싶어질 것. 5~8종/1던전으로 동결, 나머지는 백로그.
- 가변 사이클 시간 도입은 simulate의 첫 구조 변경 — 기존 17개 테스트가 회귀 방지선.
- 기대값 전투는 "운빨 한방"의 재미가 없다. 대신 전리품 난수와 던전 도전이 긴장감 담당.
  실제 플레이가 밋밋하면 Phase 5에서 연출(데미지 숫자 팝업)로 보강.
