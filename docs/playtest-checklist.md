# 플레이테스트 체크리스트 (Phase 2·3 DoD 수동 검증)

> 용도: 코드로 보장 못 하는 "체감" DoD 항목을 실플레이로 확인한다.
> 결과는 각 항목에 체크 + 한 줄 메모. 미통과/애매 항목이 밸런스·UX 피드백이 된다.
> 전부 통과하면 phase-2/phase-3 문서의 DoD를 채우고 Phase를 닫는다.
>
> 자동 검증 완료 항목 (여기서 다시 안 봄): 오프라인=온라인 정산 동일성, 모든 아이템
> 소비처 존재(경제 매트릭스), 전투 드랍의 제작 소비처 — 테스트 50개로 보장.

## 준비

```
npm run dev          # http://localhost:5173
cd packages/core
node scripts/test-save.mjs goblin    # 세션 2용
node scripts/test-save.mjs economy   # 세션 3용
```
출력 JSON을 게임의 설정 → 세이브 가져오기에 붙여넣기.
(데이터 수정 후엔 `npm run build` 먼저 — test-save는 dist 기준)

## 세션 1 — 초반 흐름 (새 캐릭터, ~20분)

설정 → 초기화 후 맨몸으로 시작.

- [ ] **"검부터 만들자" 동기가 생긴다** — 맨몸으로 슬라임 사냥 → 답답해서
      자연스럽게 채광→제련→구리 검 제작으로 향하면 통과
  - 메모:
- [ ] **위험도 배지(안전/주의/위험)가 체감과 맞다** — "주의" 몬스터에게 음식 없이
      도전 → 실제로 HP가 깎여 중단되면 통과. "위험"인데 멀쩡하면 밸런스 피드백
  - 메모:

## 세션 2 — 전투·던전 루프 (Phase 2 DoD 핵심, goblin 세이브)

- [ ] **준비 부족으로 던전에서 패배할 수 있다** — 음식 없이/낮은 장비로
      고블린 소굴 도전 → 패배해야 정상 (전략이 존재한다)
  - 메모:
- [ ] **패배 → 재정비 → 재도전 클리어 흐름이 재미있다** — "다시 가보자"는 마음이
      들면 통과, 귀찮기만 하면 피드백
  - 메모:
- [ ] **던전 보상으로 다음 티어가 열린다** — 클리어 → 미스릴 강화석 →
      미스릴 검 제작까지 이어진다
  - 메모:

## 세션 3 — 경제 순환 (Phase 3 DoD, economy 세이브)

- [ ] **골드를 실제로 쓰고 싶다** — 상점 도구 업그레이드를 봤을 때 "사고 싶다"는
      생각이 들면 통과. 가격이 와닿지 않으면 피드백
  - 메모:
- [ ] **슬롯 조합이 의미 있는 선택이다** — 낚시+요리 동시 가동: 생선이 실시간으로
      요리되고, 재료가 잠시 떨어져도 요리가 멈추지 않고 대기하면 정상
  - 메모:
- [ ] **신규 스킬이 전투에 체감되게 기여한다** —
      ① 연어구이(회복 50)를 사냥 음식으로 지정 → 말린 고기(15)보다 확실히 오래 버틴다
      ② 공격 물약 마시고 사냥 → 처치 속도 차이가 느껴진다
  - 메모:
- [ ] **버프가 자연스럽다** — 물약 마시면 상단 바에 칩 표시, 30분 안에 탭을 닫았다가
      만료 후 재접속 → 오프라인 정산이 어색하지 않다
  - 메모:
